# <a name="power-bi-app-storefront-details-tab"></a>Scheda dettagli di Power BI app storefront

Nella pagina **nuova offerta** usare la scheda **Dettagli storefront** per fornire informazioni di marketing, vendite e legali per i potenziali clienti. In questa scheda è inoltre possibile configurare la gestione per i lead generati da Azure Marketplace. Questo formato lungo è suddiviso in sei sezioni: **Dettagli dell'offerta**, dettagli dell' **elenco**, elementi di **Marketing**, assistenza **legale**, **assistenza clienti**e **gestione dei lead**.  Un asterisco (*) alla fine di un'etichetta di campo significa che il campo è obbligatorio.


## <a name="offer-details-section"></a>Sezione Offer Details (Dettagli dell'offerta)

Nella sezione **Dettagli offerta** immettere le informazioni generali sull'offerta AppSource.

![Sezione dettagli offerta della scheda Storefront Details (Dettagli vetrina)](./media/offer-details-section.png)

Nella tabella seguente vengono fornite ulteriori informazioni su questi campi. I campi obbligatori sono indicati da un asterisco (*).  

|   Campo               |   Descrizione                                                                           |
|-----------------------|-----------------------------------------------------------------------------------------|
| **Riepilogo dell'offerta\***     | Un breve scopo dell'app, che usa fino a 100 caratteri.                             |
| **Descrizione dell'offerta\*** | Descrizione dell'app, con un massimo di 3.000 caratteri. Questo campo supporta la formattazione HTML semplice. |
|   |    |


## <a name="listing-details-section"></a>Sezione Listing Details (Dettagli dell'inserzione)

La sezione relativa ai **Dettagli dell'elenco** raccoglie informazioni sul contesto dell'app: settori in cui viene in genere usato, la categoria dell'app, i prodotti compatibili e i termini di ricerca associati.

![Sezione dettagli inserzione della scheda Storefront Details (Dettagli vetrina)](./media/listing-details-section.png)

Per completare i campi di questa sezione, usare la tabella seguente.  I campi obbligatori sono indicati da un asterisco (*).
 
|   Campo                                  |   Descrizione                                                        |
| --------------                           | ---------------------                                                |
| **Settori**                           | Selezionare il settore in cui l'app si adatta meglio. Se l'app è relativa a più settori, lasciare vuoto questo campo.      |
| **Categorie\***                           | Selezionare fino a tre categorie correlate all'app.     |
| **Collegamento alla guida per l'app\***               | Fornire un URL a una pagina che offra la guida online per l'app.           |
| **Products your app works with (Max 3)** (Prodotti interrelati con l'app - al massimo 3) | Selezionare il segno più **nuovo** per creare un campo di testo. Nel campo immettere il nome di un prodotto con cui funziona l'app. Elencare fino a tre prodotti.       |
| **Search keywords (Max 3)** (Parole chiave di ricerca - al massimo 3)              | Immettere fino a tre parole chiave che saranno usate probabilmente dagli utenti per cercare l'applicazione in AppSource. Se, ad esempio, l'applicazione è chiamata "My emailing app", le parole chiave potrebbero essere **email**, **mailing**e **mail app**. |
|  |  |


## <a name="marketing-artifacts-section"></a>Sezione Marketing Artifacts (Artefatti di marketing)

Nella sezione **Marketing artefatti** caricare i materiali di personalizzazione e marketing da visualizzare in AppSource.  Questa sezione è suddivisa in quattro sottosezioni: **logo**, **video**, **documenti**e **schermate**. I logo e gli screenshot sono gli unici elementi di marketing richiesti. Tuttavia, per ottenere un migliore interesse per i clienti, è consigliabile aggiungere anche video e documenti.

![Sezione artefatti di marketing della scheda Storefront Details (Dettagli vetrina)](./media/marketing-artifacts-section.png)

Per completare i campi di questa sezione, usare la tabella seguente. I campi obbligatori sono indicati da un asterisco (*).
 
|    Campo                             |    Descrizione                                                    |
|   -----------                        |    -------------                                                  |
| *Logos* (Logo)                              |                                                                   |
| **Logo dell'offerta (formato PNG, 48x48)\***   | Caricare un logo da visualizzare nella panoramica dell'app o nei risultati della ricerca nell'app. AppSource supporta solo il formato PNG, con una risoluzione di 48 x 48 px.  |
| **Logo dell'offerta (formato PNG, 216x216)\*** | Caricare un logo da visualizzare nella pagina dei dettagli dell'app.  AppSource supporta solo il formato PNG, con una risoluzione di 216 x 216 px.  |
| *Video*                             |                                                                   |
| **Nome**                             | Immettere il nome o il titolo dell'app.                                          |
| **URL**                              | Immettere l'URL di un video ospitato in YouTube o Vimeo.                              |
| **Anteprima**                        | Aggiungere un'immagine di anteprima dell'app.  AppSource supporta solo il formato PNG, con una risoluzione di 1280 x 720 px.   |
| *Documents* (Documenti)                          | Aggiungere fino a tre documenti da visualizzare in AppSource, sotto l'intestazione **Learn more (altre informazioni** ).  |
| **Nome**                             | Immettere il nome o il titolo di un documento di supporto.                              |
| **File**                             | Caricare un file PDF.                             |
| *Schermate\**                      | Aggiungere fino a cinque schermate.                        |
| **Nome**                             | Immettere un nome o un titolo di una schermata.                                       |
| **Immagine**                            | Caricare una schermata PNG. La risoluzione deve essere 1280 x 720 px.  | 
|   |   |

I loghi caricati in [portale cloud partner](https://cloudpartner.azure.com) *non*devono:

- Usare una sfumatura. L'aspetto del logo deve essere flat.
- Includere il nome della società, il nome del marchio o altro testo. 
- Aspetto esteso.

## <a name="legal-section"></a>Sezione legale

Nella sezione relativa alle **Note legali** , fornire i due documenti legali richiesti per ogni offerta, ovvero l'informativa sulla privacy e le condizioni per l'utilizzo.

![Sezione legale della scheda Storefront Details (Dettagli vetrina)](./media/legal-section.png)

Per completare i campi di questa sezione, usare la tabella seguente:

|   Campo                |   Descrizione                           |
|------------------------|--------------------------------------   |
| **\* dell'URL dell'informativa sulla privacy** | URL per l'informativa sulla privacy pubblicata       |
| **Condizioni per l'utilizzo\***       | I criteri di utilizzo, formattati come testo normale o HTML semplice     |
|  |  |


## <a name="customer-support-section"></a>Sezione Customer support (Assistenza clienti)

Nella sezione **supporto** tecnico fornire l'URL di **supporto** per la pagina del supporto tecnico online.  Questa pagina deve fornire più opzioni di contatto, ad esempio telefono, posta elettronica e chat in tempo reale. 


## <a name="lead-management-section"></a>Sezione Lead Management (Gestione clienti potenziali)

Nella sezione relativa alla **gestione dei lead** configurare il sistema in modo da raccogliere i lead dei clienti che le offerte di AppSource generano. Per scegliere le opzioni di archiviazione per i lead, usare la tabella seguente:

|    Campo               |   Clienti potenziali destinatari                               |
|------------------------|--------------------------------------            |
|  **Nessuno**              | Non raccogliere lead. Questa è l'opzione predefinita.  |
| **Azure Blob (deprecated)** (BLOB di Azure - deprecata) | Specificare l' [archiviazione BLOB di Azure](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-overview) usando un nome di contenitore e una stringa di connessione.  Questa scelta è deprecata. Usare invece **tabella di Azure** .  |
| **Tabella di Azure**        | Specificare l' [archiviazione tabelle di Azure](https://docs.microsoft.com/azure/cosmos-db/table-storage-overview) usando una stringa di connessione.  |
| **Dynamics CRM Online** | Specificare [Dynamics 365](https://dynamics.microsoft.com/) usando un URL e le credenziali di autenticazione. |
| **Endpoint HTTPS**     | Specificare l'endpoint HTTPS usando un payload JSON.   |
| **Marketo**            | Specificare un'istanza di [Marketo](https://www.marketo.com/) usando un ID server, un ID Munchkin e un ID modulo.   |
| **Salesforce**         | Specificare [Salesforce](https://www.salesforce.com/) usando un identificatore di oggetto. |
|  |  |

Dopo aver pubblicato l'offerta, verrà convalidata la connessione al cliente potenziale e verrà automaticamente inviato un cliente potenziale di test alla destinazione specificata. Gestisci continuamente le informazioni sui lead e aggiorna tempestivamente le impostazioni per riflettere la tua attuale architettura di gestione dei clienti.
